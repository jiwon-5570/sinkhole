# 결과보고서

## 1. 프로젝트 개요

본 프로젝트는 공공데이터와 로컬 원본 파일을 기반으로 서울/수도권 지반침하 위험도를 분석하는 웹 대시보드이다. FastAPI 백엔드, SQLite 데이터베이스, 정적 HTML/CSS/JavaScript UI를 사용하며, 위험 점수 산정, 원인별 기여도 분석, 지도 기반 위치 확인, What-If 시뮬레이션, AI 설명, PDF 보고서 생성을 제공한다.

작성 기준일: 2026-06-04

## 2. 개발 목표

프로젝트의 목표는 다음과 같다.

- 공공데이터 기반 지반침하 위험 요인 통합
- 지역별 위험 점수와 위험 등급 산정
- 점수 산정 근거를 지도, 차트, AI 설명으로 제공
- 공사, 강우, 지하수, 지층, 시설물, 과거 사고 등 복합 요인 반영
- 사용자가 선택한 지역 또는 위치에 대한 리포트 생성
- 운영자가 로컬 공공데이터 파일을 추가하면 자동으로 분석에 반영

## 3. 시스템 구성

## 4. 주요 기능

### 4.1 위험도 대시보드

대시보드는 평균 위험도, 분석 지역 수, 고위험 지역 수, 모니터링 지점 수, 최근 탐지 건수를 카드 형태로 제공한다. 지도와 TOP 5 목록을 통해 우선 점검 대상도 확인할 수 있다.

### 4.2 지역/도로 분석

지역 단위 분석은 `regions`와 `feature_dataset`을 기준으로 수행된다. 도로 단위 분석 API도 구현되어 있으나, 운영 DB에 `road_segments` 데이터가 없으면 빈 목록으로 처리된다.

### 4.3 위험 요인 분해

위험 점수는 7개 요인으로 분해된다.

- 과거 지반침하
- GPR/탐사
- 시설물/노후건물
- 강우
- 지하수
- 지층/환경
- 공사 영향

각 요인은 최대 기여점이 정해져 있으며, 총점은 0~100점으로 계산된다.

### 4.4 공공데이터 적재

서울시 도로굴착 공사 파일은 다음 폴더에서 자동 감지된다.

```text
Project/backend/data/raw/public/seoul_road_excavation/
```

파일이 추가, 수정, 삭제되면 `construction_events`가 갱신되고, feature 캐시가 삭제된 뒤 위험도가 재계산된다.

국토교통부 지층정보와 시추공 정보는 지층/지하수/환경 취약성 판단에 사용된다.

### 4.5 What-If 시뮬레이션

사용자는 강우 증가, 지하수위 변화, 굴착 깊이, 공사장 거리, GPR 이상 신호, 시설물 노후 보정 등을 조정해 위험 점수 변화를 확인할 수 있다. GPR 정밀조사, 관로 보수, 배수 개선, 공사 통제, 모니터링 강화 같은 관리 조치도 시뮬레이션에 반영된다.

### 4.6 AI Chat 및 AI 리포트

AI Chat은 현재 분석 데이터와 DB 근거를 바탕으로 질문에 답한다. Gemini API 키가 없는 경우에도 일부 질문은 로컬 규칙 기반 답변으로 처리된다.

AI 리포트는 분석 결과와 원인별 기여도, 관리 권고를 텍스트와 PDF로 출력한다.

### 4.7 모니터링 지점

사용자는 최대 10개 모니터링 지점을 등록할 수 있다. 등록된 위치는 주기적으로 위험도를 갱신하고 마지막 점검 시각과 오류 상태를 저장한다.

## 6. 위험도 산정 방식

현재 위험도는 규칙 기반 점수로 산정된다.

| 요인 | 변환식 | 최대 기여점 |
| --- | ---: | ---: |
| 과거 지반침하 | 사고 건수 × 8.0 | 25 |
| GPR/탐사 | 이상 신호 수 × 12.0 | 30 |
| 시설물/노후 | 시설물 점수 × 0.25 | 15 |
| 강우 | 강우 점수 | 10 |
| 지하수 | 지하수 점수 | 7 |
| 지층/환경 | 환경 점수 | 7 |
| 공사 영향 | 공사 점수 × 0.3 | 6 |

위험 등급:

- 낮음: 30점 미만
- 보통: 30점 이상 60점 미만
- 높음: 60점 이상 80점 미만
- 매우 높음: 80점 이상

## 10. 한계와 개선 방향

현재 한계:

- 운영 점수는 규칙 기반이며 ML 모델 운영 반영은 아직 하지 않는다.
- 도로 분석 API는 구현되어 있으나 도로망 데이터가 필요하다.
- 공공데이터 API 품질과 응답 필드 변경에 영향을 받을 수 있다.
- SQLite는 단일 로컬 운영에는 적합하지만 대규모 운영에는 한계가 있다.
- AI 설명은 보조 해석이며 점수 산정 자체를 대체하지 않는다.

개선 방향:

- 도로망 데이터 적재 및 도로 단위 위험 분석 활성화
- PostgreSQL/PostGIS 이전 검토
- ML 학습용 feature snapshot과 label 구축
- 수집 데이터 품질 점수화
- 공공데이터 수집 실패 원인 UI 표시
- 리포트 지도 이미지와 데이터 품질 요약 강화

## 12. 결론

본 프로젝트는 지반침하 위험도를 단순 수치로만 제공하지 않고, 공공데이터 기반 근거와 원인별 기여도, 지도 시각화, AI 설명, PDF 리포트를 함께 제공하는 분석형 대시보드이다. 현재 구현은 로컬 실행과 발표/시연에 적합한 구조이며, 향후 도로망 데이터와 ML 모델을 추가하면 운영형 지반침하 위험 관리 시스템으로 확장할 수 있다.
