# 01_UI_제작_가이드

## 1. 목적

이 문서는 `Project/backend/app/static`에 구현된 싱크홀 위험도 대시보드 UI를 유지보수하고 확장하기 위한 기준이다. 현재 UI는 별도 프론트엔드 빌드 도구 없이 정적 `HTML/CSS/JavaScript`로 구성되며, FastAPI가 `/`와 `/static/*`에서 화면과 자산을 제공한다.

## 2. UI 기본 원칙

- 첫 화면에서 현재 위험 현황, 분석 대상, 지도, 우선 점검 대상을 바로 확인할 수 있어야 한다.
- 결과는 점수만 보여주지 않고 원인별 기여도와 근거 문장을 함께 제공한다.
- 지역/도로/위치 선택 후 2~3번의 조작 안에 분석과 리포트 생성이 가능해야 한다.
- 운영 도구 성격이 강하므로 과도한 장식보다 밀도 있고 읽기 쉬운 정보 배치를 우선한다.
- 화면에 표시되는 위치는 가능하면 좌표보다 도로명 주소 또는 행정구역명을 우선한다.
- 모든 위험 등급 색상과 문구는 일관되게 사용한다.

## 3. 현재 화면 구조

현재 UI 파일:

```text
Project/backend/app/static/index.html
Project/backend/app/static/styles.css
Project/backend/app/static/app.js
Project/backend/app/static/road-analysis.js
```

주요 화면 영역:

- 사이드바: 대시보드, 지도, TOP 5, What-If, AI Chat, 리포트, 도움말, API 문서 링크
- 상단바: 서비스 제목, 상태 표시, 분석 시각, 빠른 메뉴
- 핵심 지표 카드: 평균 위험도, 예측 지역 수, 높은 위험 지역 수, 모니터링 지점 수, 최근 탐지 건수
- 분석 컨트롤: 분석 단위, 지역, 도로, 분석일, 위치 검색, 지도 선택
- 지도 패널: 자체 SVG 지도 또는 Google 지도 임베드
- 선택 결과 패널: 위험 점수, 등급, 우선순위, 도로명 주소, 기여 요인
- TOP 5 패널: 우선 점검 대상 목록
- 추이/분포/과거 이력 패널: 위험도 추이, 등급 분포, 지반침하 원인 분포
- AI 원인 설명 카드: 위험 점수 산정 근거 요약
- 리포트 패널: PDF 생성 결과, 파일 목록, 열기/다운로드/삭제
- What-If Drawer: 강우, 굴착, 지하수, GPR 이상 신호, 관리 조치 적용
- AI Chat Dialog: 현재 분석 데이터 기반 질의응답
- Metric Detail Dialog: 지표 상세 근거와 관리 방안

## 4. 주요 사용자 흐름

### 4.1 지역 분석

1. 대시보드 진입
2. 분석 단위에서 `지역` 선택
3. 지역과 분석일 선택
4. `위험도 분석` 실행
5. 지도, 점수, 기여 요인, 과거 이력, AI 설명 확인
6. 필요 시 `AI 리포트 생성`

### 4.2 도로 분석

1. 분석 단위에서 `도로` 선택
2. 지역 선택 후 도로 목록 로드
3. 도로와 분석일 선택
4. `위험도 분석` 실행
5. 도로 기준 위험 점수와 근거 확인

현재 운영 DB에는 도로 세그먼트가 없을 수 있다. 이 경우 UI는 도로 선택 결과가 비어 있는 상태를 정상적으로 처리해야 한다.

### 4.3 위치 직접 분석

1. 빠른 메뉴 또는 도움말에서 실시간 모드 선택
2. 주소, 장소명 또는 `위도,경도` 입력
3. `지도에 적용` 또는 `지도에서 선택`
4. 주변 분석 지점과 상업/현장 관점 위험 정보 확인

### 4.4 What-If 시뮬레이션

1. 사이드바에서 What-If 열기
2. 프리셋 또는 수동 값 선택
3. 강우량, 지하수위 변화, 굴착 깊이, 공사장 거리, GPR 이상 신호, 시설물 노후 보정 입력
4. 관리 조치 적용 여부 선택
5. `시뮬레이션 실행`
6. 기존 점수, 시나리오 점수, 관리 후 점수, 주요 변화 요인 확인

## 5. 위험 등급 표시 기준

UI와 백엔드는 같은 등급 기준을 사용한다.

| 등급 | 점수 | UI 의미 |
| --- | ---: | --- |
| 낮음 | 0 이상 30 미만 | 일반 관찰 |
| 보통 | 30 이상 60 미만 | 정기 점검 |
| 높음 | 60 이상 80 미만 | 우선 점검 |
| 매우 높음 | 80 이상 | 즉시 현장 확인 |

색상은 `riskMeta()`와 CSS 클래스를 기준으로 통일한다.

## 6. 패널별 설계 기준

### 6.1 핵심 지표 카드

- 숫자는 크고 짧게 표시한다.
- 상세 정보가 있는 카드는 클릭 가능해야 한다.
- 모니터링 지점 카드에서는 등록/갱신/해제를 지원한다.

### 6.2 지도 패널

- 지역 점수는 색상과 크기로 구분한다.
- 선택 위치는 별도 표시한다.
- Google 지도 API 키가 노출되지 않거나 임베드가 실패해도 자체 지도 또는 fallback 링크를 제공한다.
- 지도 클릭 선택 모드는 명확한 시작/종료 상태를 가져야 한다.

### 6.3 과거 이력 패널

- `지반침하 원인 분포`는 선택 지역의 원인 유형별 건수를 보여준다.
- 선택 지역 데이터가 부족하면 전체 지역 기반 fallback임을 문구로 알려야 한다.
- 과거 이력은 현재 침하 발생을 의미하지 않으므로 "반복 취약성 참고 지표"로 표현한다.

### 6.4 기여 요인 패널

기여 요인은 다음 7개 축으로 표시한다.

- 과거 지반침하
- GPR/탐사
- 시설물/노후건물
- 강우
- 지하수
- 지층/환경
- 공사 영향

값은 백엔드 `breakdown` 응답을 그대로 사용한다.

### 6.5 리포트 패널

- 리포트 생성 직후 본문 요약을 표시한다.
- 생성된 PDF 목록은 체크박스로 다중 선택 가능해야 한다.
- 열기, 다운로드, 삭제는 선택된 항목 기준으로 동작한다.
- 삭제는 `/api/reports/delete`를 호출한다.

## 7. API 연동 기준

UI에서 사용하는 주요 API:

```text
GET  /api/health
GET  /api/app-config
GET  /api/summary
GET  /api/regions
GET  /api/roads
POST /api/analyze-risk
POST /api/analyze-road-risk
GET  /api/top-risk-regions
GET  /api/top-risk-roads
GET  /api/charts/risk-distribution
GET  /api/charts/risk-trend
GET  /api/charts/sinkhole-cause-distribution
POST /api/simulate-risk
POST /api/ai-chat
POST /api/generate-report
GET  /api/reports
POST /api/reports/delete
GET  /api/monitoring-points
POST /api/monitoring-points
POST /api/monitoring-points/refresh
DELETE /api/monitoring-points/{point_id}
```

모든 API 호출은 `app.js`의 `api()` helper를 통해 처리한다. 실패 시 화면에 짧은 상태 메시지를 표시하고 UI가 빈 상태로 유지되도록 한다.

## 8. 반응형 기준

- 데스크톱: 사이드바 + 본문 그리드 구조 유지
- 태블릿: 패널을 1~2열로 재배치
- 모바일: 지도, 분석 컨트롤, 결과 패널 순서로 세로 배치
- 버튼 텍스트가 줄바꿈되더라도 높이 변화가 과도하지 않아야 한다.
- 표는 가로 스크롤 또는 카드형 요약으로 대체한다.

## 9. 문구 기준

- 결과 문구는 "위험 가능성", "점검 우선순위", "근거" 중심으로 작성한다.
- "현재 침하가 발생 중"처럼 실제 탐지로 오해될 수 있는 문구를 피한다.
- 데이터가 부족한 경우 "데이터 없음"보다 "현재 연동된 데이터 기준으로 산출할 수 없습니다"처럼 이유를 명확히 쓴다.
- AI 설명은 보조 해석이며 점수 산정 자체는 백엔드 계산 결과임을 유지한다.

## 10. 유지보수 체크리스트

- [ ] 신규 API 응답 필드는 `renderResult()`와 관련 렌더러에서 null 안전 처리
- [ ] 위험 등급 기준 변경 시 백엔드와 UI 동시 갱신
- [ ] 지도 fallback 동작 확인
- [ ] 리포트 목록 선택/삭제 동작 확인
- [ ] What-If 결과가 빈 배열일 때 빈 상태 표시
- [ ] 모니터링 지점 최대 개수 제한 표시
- [ ] 한국어/영어 언어 선택 후 주요 텍스트 갱신 확인
