# 10_배포_및_운영_가이드

## 1. 목적

이 문서는 싱크홀 위험도 대시보드의 로컬 실행, 운영 설정, 데이터 관리, 장애 대응, 배포 기준을 설명한다.

## 2. 로컬 설치

처음 설치:

```powershell
.\Install_Sinkhole.cmd
```

설치 스크립트가 수행하는 작업:

- `Project/backend/.venv` 생성
- `requirements.txt` 설치
- DB 스키마 초기화
- 루트/바탕화면 실행 바로가기 생성
- 설치 후 앱 실행

## 3. 로컬 실행

일반 실행:

```powershell
.\Run_Sinkhole.cmd
```

기본 URL:

```text
http://127.0.0.1:5000
```

수동 실행:

```powershell
cd .\Project\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 5000
```

## 4. 환경 변수

설정 파일:

```text
Project/backend/.env
```

필수/권장 값:

```env
PUBLIC_DATA_API_KEY=...
SEOUL_OPEN_DATA_API_KEY=...
SAFEMAP_API_KEY=...
GEMINI_API_KEY=...
GOOGLE_MAPS_API_KEY=...
SINKHOLE_EXPOSE_GOOGLE_MAPS_KEY=0
SINKHOLE_SEED_DEMO=0
```

운영 설정:

```env
SINKHOLE_ENV=production
SINKHOLE_RELOAD=0
SINKHOLE_ENABLE_BASIC_AUTH=1
SINKHOLE_BASIC_AUTH_USERNAME=...
SINKHOLE_BASIC_AUTH_PASSWORD=...
SINKHOLE_PUBLIC_DATA_AUTO_COLLECT=1
SINKHOLE_PUBLIC_DATA_COLLECT_ON_START=1
SINKHOLE_LOCAL_CONSTRUCTION_FILE_IMPORT_ENABLED=1
```

## 8. 로그

실행 스크립트 로그:

```text
Project/backend/data/tmp/launcher.out.log
Project/backend/data/tmp/launcher.err.log
```

다른 실행 스크립트도 `data/tmp` 아래에 로그를 남길 수 있다.

확인 항목:

- 서버 시작 실패
- 포트 충돌
- public data 수집 오류
- AI API timeout
- 파일 적재 오류

## 9. 백업

운영 전 백업 권장 항목:

- `Project/backend/db/app.db`
- `Project/backend/.env`
- `Project/backend/data/raw/public/`
- `Project/backend/data/reports/`

대용량 DB는 파일 복사 전 서버를 중지하거나 SQLite 백업 방식을 사용한다.

## 10. 장애 대응

| 증상 | 확인 |
| --- | --- |
| 서버가 열리지 않음 | `data/tmp/*.err.log`, 포트 5000 사용 여부 |
| API 401 | Basic Auth 설정 |
| 지도 키 노출 안 됨 | `GOOGLE_MAPS_API_KEY`, `SINKHOLE_EXPOSE_GOOGLE_MAPS_KEY` |
| AI 답변 품질 낮음 | `GEMINI_API_KEY`, 로컬 근거 데이터 |
| 위험 점수 0점 | 데이터 적재 상태, feature 캐시 |
| 도로 목록 없음 | `road_segments` 데이터 유무 |
| 도로굴착 반영 안 됨 | 파일 위치, 인코딩, `local_file_import_state` |
| 공공데이터 수집 실패 | API 키, 네트워크, 상태 API 오류 메시지 |

## 11. 보안 기준

- API 키는 `.env`에만 저장한다.
- 운영 환경에서는 Basic Auth를 활성화한다.
- Google Maps 키는 필요할 때만 프론트에 노출한다.
- 로그에 API 키가 남지 않도록 redaction 로직을 유지한다.
- PDF와 DB 파일은 외부 공유 전에 민감 정보 포함 여부를 확인한다.

## 12. Cloud Run 배포 참고

백엔드 폴더에는 Cloud Run 관련 파일이 있다.

```text
Project/backend/Dockerfile
Project/backend/Deploy_Cloud_Run.ps1
Project/backend/README_CLOUD_RUN.md
Project/backend/cloudrun.env.example
```

Cloud Run 배포 시 고려사항:

- SQLite 파일은 컨테이너 재시작 시 영속성이 제한될 수 있다.
- 운영 DB를 유지하려면 외부 DB 또는 영속 스토리지 전략이 필요하다.
- API 키는 Secret Manager 또는 환경 변수로 주입한다.
- 보고서 PDF 저장소도 별도 스토리지가 필요하다.

## 14. 변경 배포 기준

- 위험 점수 로직 변경은 문서, UI 문구, 리포트 문구를 함께 갱신한다.
- 스키마 변경 전 DB 백업을 남긴다.
- 공공데이터 수집 로직 변경 후 상태 API와 적재 건수를 확인한다.
- UI 변경 후 분석, 리포트, What-If, 모니터링 주요 흐름을 수동 점검한다.
