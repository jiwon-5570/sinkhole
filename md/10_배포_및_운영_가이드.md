# 국토안전관리 프로젝트 배포 및 운영 가이드

## 1. 문서 목적

이 문서는 국토안전관리 프로젝트를 실행, 시연, 배포, 운영하기 위한 기준을 정의한다.

본 가이드는 다음을 목표로 한다.

1. 개발 완료된 프로젝트를 실제 실행 가능한 상태로 만든다.
2. 공모전 시연 환경을 안정적으로 구성한다.
3. 향후 서비스 확장을 고려한 배포 구조를 제시한다.
4. API, DB, AI 모델, 프론트엔드까지 전체 시스템 운영 기준을 정의한다.

---

## 2. 전체 배포 구조

프로젝트는 다음과 같은 구조로 배포된다.

```text
[사용자]
   ↓
[Frontend (React)]
   ↓
[Backend API (Flask/FastAPI)]
   ↓
[DB (SQLite / PostgreSQL)]
   ↓
[ML Model (XGBoost)]
   ↓
[AI API (Gemini)]

4. 프로젝트 실행 방법

기본 주소:

→ API 호출
→ 화면 출력
5. 환경 변수 설정
5.1 Gemini API Key

중요:
API 키는 절대 코드에 직접 넣지 않는다.

방법
export GEMINI_API_KEY=AIzaSyBZ0L1XKsngHK4jA7_WqBk995qcrXAUNuI

또는 .env 파일 사용

GEMINI_API_KEY=AIzaSyBZ0L1XKsngHK4jA7_WqBk995qcrXAUNuI
5.2 Backend에서 사용
import os

api_key = os.getenv("GEMINI_API_KEY")
