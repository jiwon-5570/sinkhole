# 국토안전관리 프로젝트 예측 알고리즘 가이드

## 1. 문서 목적

이 문서는 국토안전관리 프로젝트에서 사용할 예측 알고리즘의 목적, 선정 기준, 후보 모델, 최종 선정안, 학습 방식, 평가 방법, 설명 가능성 확보 전략을 정의한다.

본 가이드는 다음을 목표로 한다.

1. 지반침하 위험 예측에 적합한 모델을 선정한다.
2. 데이터 구조에 맞는 알고리즘을 선택한다.
3. 공모전 수준에서 구현 가능한 현실적인 모델 구조를 제시한다.
4. 모델 결과를 지도, 그래프, 설명 리포트로 연결할 수 있도록 한다.
5. “예측”과 “설명”의 역할을 분리하여 설계한다.

---

## 2. 예측 목표 정의

이 프로젝트의 모델 목표는 **정확한 싱크홀 발생 시점 예언**이 아니다.  
지반침하 및 싱크홀은 국지적이고 복합적인 현상이므로, 실제 서비스 목표는 아래와 같이 정의한다.

### 2.1 1차 목표
지역별 **지반침하 위험 점수**를 산정한다.

예시:
- 0 ~ 100 점수

### 2.2 2차 목표
위험 점수를 바탕으로 **위험 등급**을 분류한다.

예시:
- 낮음
- 보통
- 높음
- 매우 높음

### 2.3 3차 목표
단순 위험 점수 외에 **위험 상승 신호**를 탐지한다.

예시:
- 최근 7일 강우량 증가로 단기 위험 상승 가능성
- 최근 지하수위 변동 증가
- GPR 탐사 결과 기반 취약성 보강

## 3. 알고리즘 선택 원칙

이번 프로젝트의 예측 알고리즘은 아래 기준을 만족해야 한다.

### 3.1 정형 데이터에 강해야 한다
입력 데이터는 대부분 아래와 같은 정형·공간 feature 형태다.
- 과거 사고 이력
- 지반침하 점검 이력
- GPR 탐사 결과
- 강우량
- 지하수위
- 주변 환경 점수

따라서 이미지 전용 딥러닝보다 **트리 기반 모델**이 적합하다.

### 3.2 설명 가능성이 높아야 한다
공모전에서는 “왜 이 결과가 나왔는가”를 설명할 수 있어야 한다.

따라서 다음이 가능해야 한다.
- feature importance
- 변수 기여도
- SHAP 기반 설명

### 3.3 소규모 ~ 중규모 데이터셋에도 강해야 한다
현실적으로 데이터 정제 후 학습 가능한 표본 수는 무제한이 아니다.  
따라서 적은 데이터에서도 안정적으로 동작하는 모델이 유리하다.

### 3.4 규칙 기반 점수와 결합 가능해야 한다
초기 MVP에서는 완전한 ML보다 **규칙 기반 + ML 고도화** 구조가 현실적이다.  
따라서 이 구조와 자연스럽게 결합 가능해야 한다.

## 4. 데이터 구조와 모델 적합성

## 4.1 입력 데이터 형태
이번 프로젝트에서 모델 입력은 아래와 같은 구조다.

```text
region_id | analysis_date | feature_1 | feature_2 | ... | target
feature 예시:

past_sinkhole_count
gpr_detected_count
rainfall_7d
groundwater_variation
building_density
facility_aging_score

즉, 데이터 구조는 표 형태(tabular) 가 중심이다.

4.2 공간 데이터 처리 방식

지도 기반 서비스라고 해서 반드시 공간전용 딥러닝을 써야 하는 것은 아니다.

이번 프로젝트에서는 공간 데이터를 다음과 같이 처리한다.

방법
격자(grid) 단위 또는 행정동 단위로 분석 단위 설정
각 단위별 주변 데이터를 집계
집계된 값을 feature로 변환

예시:

반경 500m 내 과거 사고 수
반경 300m 내 GPR 탐지 건수
해당 grid 내 건물 밀집도
해당 지역 최근 7일 강우량

즉, 공간 데이터를 정형 feature로 변환하여 모델 입력으로 사용한다.

5. 모델 후보군
5.1 규칙 기반 위험도 산정 모델
개념

전문가 규칙 또는 가중치 기반으로 위험 점수를 계산

예시:

과거 사고 이력 많음: +20
GPR 공동 탐지 있음: +25
최근 7일 강우량 높음: +10
지하수위 변동 큼: +8
시설물 노후도 높음: +12
장점
데이터가 적어도 바로 구현 가능
설명이 매우 쉬움
공모전 MVP에 적합
지도와 그래프 생성에 바로 사용 가능
단점
학습 기반 최적화 한계
일반화 성능 한계
결론

초기 MVP 필수 모델
ML 모델 전에 반드시 유지하는 것이 좋다.

5.2 Random Forest
개념

여러 개의 의사결정나무를 앙상블하여 예측

장점
구현 쉬움
baseline 모델로 적합
과적합에 비교적 강함
feature importance 추출 가능
단점
최종 성능은 부스팅 계열보다 낮을 수 있음
위험도 미세 차이 반영에 한계 가능
결론

비교용 baseline 모델로 적합

5.3 XGBoost
개념

Gradient Boosted Decision Trees 기반의 대표적인 고성능 모델

장점
정형 데이터 예측에 매우 강함
중소규모 데이터에서도 성능 우수
결측값 처리에 유리
회귀/분류 모두 가능
feature importance 및 SHAP 연계 용이
공모전 발표에서 설명하기 좋음
단점
파라미터 튜닝 필요
학습 파이프라인 설계 필요
결론

이번 프로젝트의 주력 예측 모델로 가장 적합

5.4 LightGBM
개념

트리 기반 gradient boosting framework

장점
빠른 학습 속도
대규모 데이터에 효율적
sparse 데이터에 유리
회귀/분류 가능
단점
데이터가 많지 않을 경우 XGBoost 대비 절대 우세라고 보기 어려움
튜닝과 검증 필요
결론

비교 모델로 채택
XGBoost와 성능 비교용으로 적절

5.5 CatBoost
개념

범주형 변수 처리를 강점으로 하는 boosting 모델

장점
범주형 변수 자동 처리 강점
지반 유형, 사고 원인, 토지이용 등 categorical feature가 많을 때 유리
단점
이번 프로젝트에서 필수라고 보긴 어려움
팀이 다뤄본 경험이 적다면 오히려 부담
결론

선택 가능하나, 이번 프로젝트에서는 우선순위 낮음

5.6 CNN / 딥러닝 계열
개념

이미지나 시계열을 직접 입력으로 활용하는 딥러닝 모델

적용 가능 영역
GPR 단면 이미지
CCTV 기반 위험 탐지
비정형 이미지 학습
장점
이미지 직접 분석 가능
단점
이번 프로젝트의 중심 과제는 지도 기반 위험도 서비스
구현 복잡도 높음
설명성 낮음
공모전 시연용으로 부담 큼
결론

주력 모델로는 비추천
향후 확장 연구용으로만 유지

7. 예측 구조 설계
7.1 내부 구조

모델은 아래 구조로 작동한다.

원천 데이터
→ Feature 생성
→ 규칙 기반 점수 산정
→ XGBoost 학습/예측
→ risk_score 출력
→ risk_level 변환
→ SHAP 설명 생성
→ Gemini 설명 리포트 생성
7.2 출력 구조
출력 1: risk_score
연속형 값
0 ~ 100 범위
출력 2: risk_level

risk_score를 등급으로 변환

예시:

0 ~ 29: 낮음
30 ~ 59: 보통
60 ~ 79: 높음
80 ~ 100: 매우 높음
출력 3: risk_change_signal

최근 변화 기반 위험 상승 신호

예시:

stable
increasing
rapidly_increasing

규칙 기반 위험도 모델로 초기 분석 구조를 만든다.

Gemini API는 예측이 아니라 설명 리포트 생성에 사용한다.

즉, 이 프로젝트의 알고리즘 구조는
“정형 데이터 기반 위험 예측 + 설명 가능한 분석 + 생성형 AI 리포트”
구조로 정의된다.
