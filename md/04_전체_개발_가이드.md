# 04_전체_개발_가이드

## 1. 목적

이 문서는 싱크홀 위험도 대시보드의 전체 개발 구조, 실행 흐름, 주요 모듈, 작업 우선순위를 정리한다. 현재 구현은 FastAPI 백엔드, SQLite DB, 정적 HTML/CSS/JavaScript UI로 구성된다.

## 2. 최종 목표

서비스의 목표는 공공데이터와 로컬 원본 파일을 기반으로 서울/수도권 지반침하 위험도를 계산하고, 사용자가 지도/차트/AI 설명/PDF 리포트로 위험 근거를 확인하도록 지원하는 것이다.

핵심 산출:

- 지역별 위험 점수와 등급
- 우선 점검 TOP 5
- 위험 기여 요인 분해
- 과거 이력과 원인 분포
- What-If 시뮬레이션 결과
- AI 채팅과 AI 리포트
- PDF 보고서
- 모니터링 지점 관리

## 3. 현재 기술 스택

| 영역 | 현재 구현 |
| --- | --- |
| Backend | FastAPI |
| Server | Uvicorn |
| DB | SQLite |
| Data | pandas, openpyxl, requests, pyproj |
| Scoring | 규칙 기반 점수 계산 |
| AI | Gemini API 선택 연동 |
| PDF | reportlab |
| Frontend | 정적 HTML/CSS/JavaScript |
| 지도 | 자체 SVG 지도, Google Maps embed fallback |
| 실행 | Windows CMD/PowerShell 스크립트 |

`requirements.txt`에는 XGBoost/SHAP도 포함되어 있으나, 현재 운영 위험 점수는 `risk_scoring.py`의 규칙 기반 계산을 사용한다.

## 6. 주요 모듈 책임

### 6.1 Routes

- `analysis.py`: 지역/도로 위험 분석
- `regions.py`, `roads.py`: 분석 대상 조회
- `map.py`: geocode, reverse geocode, 지도 layer
- `charts.py`: 위험 분포, 추이, 원인 분포
- `compare.py`: 지역 비교
- `simulation.py`: What-If 시뮬레이션
- `chat.py`: AI Chat 응답
- `report.py`: AI 리포트와 PDF 관리
- `public_data.py`: 공공데이터 수집/상태
- `monitoring.py`: 모니터링 지점 관리
- `commercial.py`: 위치 직접 분석/현장형 리포트
- `health.py`: 상태 확인

### 6.2 Services

- `features.py`: 위험 feature 생성
- `risk_scoring.py`: 점수/등급 계산
- `ground_layers.py`: 지층 요약과 환경 보정
- `public_data_collector.py`: 공공데이터포털/KALIS/MOLIT/KMA 수집
- `seoul_open_data_collector.py`: 서울 열린데이터 수집
- `local_construction_importer.py`: 로컬 도로굴착 파일 적재
- `monitoring_points.py`: 모니터링 지점 갱신
- `simulation_engine.py`: What-If 점수 변화 계산
- `reporting.py`: 리포트 텍스트 생성
- `report_pdf.py`: PDF 렌더링
- `ai_evidence.py`: AI 답변용 근거 조회
- `commercial.py`: 주소/좌표 기반 현장형 분석

## 7. 데이터 처리 흐름

```text
공공데이터 API / 로컬 파일
-> 원본 저장 또는 파일 상태 기록
-> 정규화 테이블 반영
-> feature 생성
-> 규칙 기반 점수 계산
-> 위험 결과 저장
-> API 응답
-> UI 렌더링 / PDF 생성 / AI 설명
```

## 11. 개발 규칙

- 운영 DB와 `.env`는 Git에 포함하지 않는다.
- 사용자 또는 운영자가 추가한 원본 파일을 임의 삭제하지 않는다.
- 공공데이터 수집 실패는 UI 전체 장애로 이어지지 않게 한다.
- 새 API는 `ok()`/`fail()` 응답 형식을 맞춘다.
- 위험 점수 기준 변경 시 `risk_scoring.py`, UI 문구, 리포트 문구를 함께 갱신한다.
- 문서와 README는 실제 구현 기준으로 유지한다.

## 12. 확장 로드맵

- 도로망 데이터를 확보해 `road_segments` 기반 도로 분석을 활성화한다.
- ML 모델 학습 파이프라인을 도입해 규칙 점수와 비교한다.
- PostgreSQL/PostGIS 이전을 검토한다.
- API 수집 실패 원인을 UI에서 관리자에게 표시한다.
- PDF 리포트에 지도 이미지와 데이터 품질 요약을 강화한다.
- 모니터링 지점의 주기 알림 기능을 추가한다.
