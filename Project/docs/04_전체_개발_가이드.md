# 국토안전관리 프로젝트 전체 개발 가이드

## 1. 문서 목적

이 문서는 국토안전관리 프로젝트 전체 개발 흐름을 정의하는 문서이다.

본 가이드는 다음을 목표로 한다.

1. 프로젝트의 전체 구조를 한 번에 이해할 수 있도록 한다.
2. 백엔드, 프론트엔드, DB, 모델 학습 영역을 일관된 흐름으로 연결한다.
3. 실제 VSCode 기반 개발을 바로 시작할 수 있도록 폴더 구조와 작업 순서를 제시한다.
4. 공모전용 프로토타입과 향후 확장형 서비스의 구분을 명확히 한다.

---

## 2. 프로젝트 최종 목표

이 프로젝트의 최종 목표는 다음과 같다.

- 국토안전관리원 지반침하 및 지하안전 데이터를 중심으로 지역별 기본 취약도를 분석한다.
- 기상, 지하수, 환경 데이터를 결합해 위험 상승 가능성을 계산한다.
- 결과를 지도와 그래프로 시각화한다.
- 우선 점검 대상지를 추천한다.
- 생성형 AI를 활용해 위험 원인과 점검 권고안을 자연어로 설명한다.

즉, 이 프로젝트는 단순 조회 시스템이 아니라  
**분석 + 시각화 + 설명 + 의사결정 지원**을 결합한 서비스다.

14. 발표용 구현 전략

공모전에서는 완성도보다 보여주는 흐름이 중요하다.

따라서 구현 전략은 다음과 같다.

더미가 아니라 실제 구조처럼 보이는 데이터 사용
지도 화면이 반드시 먼저 보이게 구성
고위험 지역 클릭 → 상세 분석 → 그래프 → AI 리포트 흐름 구성
“왜 위험한가”가 한 화면 안에서 설명되도록 구성
15. 개발 시 주의사항
공사 데이터가 중심처럼 보이면 안 된다.
예측 모델은 생성형 AI가 아니라 정형 데이터 모델이 담당해야 한다.
설명형 AI는 결과 해석용으로만 사용한다.
데이터 부족을 고려하여 초기에는 규칙 기반 구조도 함께 유지한다.
서비스 결과는 반드시 지도와 그래프로 제시한다.
DB, API, UI가 서로 다른 논리로 설계되지 않도록 통일한다.
16. 최종 요약

이 프로젝트의 전체 개발 구조는 다음 4단계로 요약된다.

데이터 구축
분석/예측 구축
API 및 UI 구축
설명형 AI 및 시연 정리

즉, 이 프로젝트의 핵심은 단순 모델 구현이 아니라
국토안전관리원 데이터 기반 위험 분석 결과를 실제 서비스 형태로 보여주는 것이다.
