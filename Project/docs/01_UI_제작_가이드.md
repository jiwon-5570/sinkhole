# 국토안전관리 프로젝트 UI 제작 가이드

## 1. 문서 목적

이 문서는 **국토안전관리원 공공데이터 기반 지반침하 위험 분석 및 우선점검 지원 서비스**의  
UI/UX 설계, 화면 구조 정의, 컴포넌트 구성, 메뉴 트리, 시각화 방식, API 연동 방향을 정리한 문서이다.

이 문서의 목적은 다음과 같다.

1. 서비스의 전체 화면 구조를 일관되게 설계한다.
2. 프론트엔드 개발자가 화면을 실제로 구현할 수 있도록 충분한 기준을 제공한다.
3. 발표용 시연 화면까지 고려하여 시각적으로 설득력 있는 구조를 만든다.
4. 국토안전관리원 데이터 중심의 서비스라는 정체성이 UI에서도 명확히 드러나도록 한다.

---

## 2. 서비스 UI의 기본 철학

이 서비스는 단순한 조회 시스템이 아니다.  
**지도 기반 위험도 시각화 + 데이터 그래프 분석 + 설명형 AI 리포트**를 결합한  
국민/지자체/실무 담당자용 분석 서비스다.

따라서 UI는 다음 원칙을 반드시 따라야 한다.

### 2.1 지도 중심 구조
지반침하 위험은 공간성과 지역성이 강하므로,  
사용자는 먼저 **“어디가 위험한가”**를 지도에서 직관적으로 이해할 수 있어야 한다.

### 2.2 데이터 근거 중심 구조
단순히 위험 등급만 보여주지 않고,  
왜 그런 결과가 나왔는지 그래프와 수치로 보여줘야 한다.

### 2.3 설명 가능한 구조
사용자는 결과를 보고 다음 질문을 하게 된다.
- 왜 이 지역이 위험한가?
- 무엇이 영향을 줬는가?
- 지금 무엇을 해야 하는가?

따라서 UI는 **설명 패널 / 리포트 영역**을 반드시 포함해야 한다.

### 2.4 발표와 실사용을 동시에 고려
이 서비스는 공모전 제출 및 시연도 중요하므로,
- 첫 화면에서 서비스 가치가 바로 보여야 하고
- 2~3번 클릭 내에 핵심 결과가 나타나야 하며
- 그래프와 지도 화면이 시각적으로 정돈되어 있어야 한다.

### 2.5 국토안전관리원 데이터 중심성 반영
UI 문구와 화면 구조에서 **국토안전관리원 데이터 기반 서비스**라는 점이 드러나야 한다.
공사예정 데이터는 보조 표현만 허용하며 중심처럼 보이면 안 된다.

## 3. 사용자 정의

이 서비스의 주요 사용자는 3종류로 구분한다.

### 3.1 일반 사용자
목적:
- 내가 사는 지역이 위험한지 확인
- 지도와 간단한 그래프를 통해 현재 위험도를 확인
- 이해하기 쉬운 설명 리포트 확인

UI 특성:
- 직관적
- 설명 중심
- 쉬운 용어 사용

### 3.2 지자체/실무 담당자
목적:
- 점검 우선순위 확인
- 지역별 위험도 비교
- 최근 위험 상승 지역 확인
- 근거 데이터와 점검 권고 확인

UI 특성:
- 지도 + 표 + 그래프 조합
- 필터 기능 중요
- 비교 분석 기능 중요

### 3.3 공모전 심사위원 / 발표 청중
목적:
- 서비스 구조를 빠르게 이해
- 데이터 활용과 AI 활용이 실제로 어떻게 이루어지는지 확인
- 구현 가능성과 차별성 확인

UI 특성:
- 첫 화면 임팩트 필요
- 지도와 요약 카드 중요
- 발표용 흐름이 자연스러워야 함

## 4. 전체 UI 구조 개요

서비스는 아래 6개 핵심 화면으로 구성한다.

1. 대시보드
2. 위험 예측 지도
3. 데이터 분석
4. 지역 상세 분석
5. 지역 비교 분석
6. 리포트 출력

이 6개 화면을 기준으로 메뉴를 구성한다.

14. UI 구현 시 주의사항
공사 데이터가 화면의 중심처럼 보이면 안 된다.
결과는 반드시 지도와 그래프로 함께 제시한다.
사용자는 2~3번 클릭 내에 고위험 지역과 이유를 확인할 수 있어야 한다.
국토안전관리원 데이터 기반이라는 정체성이 문구와 구조에서 드러나야 한다.
점검 추천은 단순 점수보다 “이유”까지 함께 표시해야 한다.
발표용 시연 흐름이 화면 구조에서 자연스럽게 이어져야 한다.

17. 최종 요약

이 서비스의 UI는 다음 3축으로 설계한다.

지도 기반 위험도 시각화
그래프 기반 분석 근거 제시
설명형 AI 리포트 제공

즉, 이 UI는 단순 조회 화면이 아니라
국토안전관리원 데이터 기반으로 지반침하 위험을 설명하고, 우선 점검 의사결정을 지원하는 분석형 UI여야 한다.
