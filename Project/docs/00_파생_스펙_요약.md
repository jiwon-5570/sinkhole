# 파생 스펙 요약 (md/ 10개 문서 기반)

이 문서는 `project/docs/01~10_*.md`의 내용을 “바로 코드로 옮길 수 있는 형태”로 요약한 것이다.

## 목표(서비스)

- 지역(`region_id`) 단위로 지반침하 위험도를 산정/예측하고,
- 지도(레이어), 그래프(근거), 우선점검(순위), 리포트(설명)를 제공한다.
- 핵심 흐름: 데이터 → feature_dataset → 위험도/등급/순위 → API → UI → (선택) AI 설명.

## UI(핵심 화면/메뉴)

`01_UI_제작_가이드.md` 기준 메뉴 트리(요약):
- 대시보드(전체 현황, TOP5, 최근상승, 요약그래프)
- 위험 예측 지도(검색/필터/레이어/상세패널)
- 데이터 분석(분포/추이/기여도/순위표)
- 지역 상세 분석(점수카드, 변화추이, 점검사유, AI 리포트)
- 지역 비교 분석
- 리포트(PDF/발표용)

## 데이터/Feature(요약)

`02_공공데이터_학습_가이드.md`, `07_Feature_엔지니어링_가이드.md` 기준:
- 데이터는 지역 단위로 매핑하고(권장: grid), 시간 기준 집계 후 feature로 변환.
- feature 그룹:
  - 기본 취약도: 과거사고/시설물/지반/원인 등
  - GPR/탐사: 공동탐지 여부/건수/점검 빈도
  - 기상: 강우(1/3/7/30일), 온습도 변화
  - 지하수: 수위/변동성
  - 환경/보조: 건물/도로 밀도, 토지이용, 공사(보조 가중치)

## DB(프로토타입 기준)

`03_DB_제작_가이드.md` 기준 핵심 테이블:
- `regions` (기준)
- raw 성격: `sinkhole_history`, `gpr_inspection`, `facility_safety`, `weather_data`, `groundwater_data`, `environment_features`, `construction_events`
- feature: `feature_dataset` (region_id + analysis_date PK)
- 결과: `risk_analysis_result`

추가(운영 가이드에 언급): 캐싱용 `ai_report` 테이블을 별도로 둔다.

## 알고리즘/출력(요약)

`05_예측_알고리즘_가이드.md` 기준:
- 1단계(필수): 규칙 기반 점수(가중치 합산)
- 2단계(주력): XGBoost(회귀→score) + 등급화
- 설명: SHAP(변수 기여), 리포트(선택: Gemini)
- 등급 예시:
  - 0~29 낮음 / 30~59 보통 / 60~79 높음 / 80~100 매우 높음

## API(초기 구현 대상)

`06_API_설계_가이드.md` 기준(핵심 엔드포인트):
- `GET /api/health`
- `GET /api/regions`
- `GET /api/region/{region_id}`
- `GET /api/map/risk-layer`
- `GET /api/map/gpr-layer`
- `GET /api/map/hotspots`
- `POST /api/analyze-risk`
- `GET /api/analysis/{region_id}`
- `GET /api/top-risk-regions`
- `GET /api/charts/risk-distribution`
- `GET /api/charts/risk-trend`
- `GET /api/charts/factor-importance`
- `GET /api/charts/top-priority`
- `POST /api/compare-regions`
- `POST /api/generate-report`
- `GET /api/report/{region_id}`

공통 응답 규칙:
```json
{"success": true, "message": "…", "data": {}}
```

## 배포/운영(요약)

`10_배포_및_운영_가이드.md` 기준:
- 개발 단계: SQLite, 로컬 실행(공모전 시연 안정성)
- 확장 단계: PostgreSQL(+PostGIS)
- 운영 포인트: 결과 캐싱, 로그, 장애 시 템플릿 리포트 fallback, API 키(.env)로 관리
